Signal & Decision Glossary
BUY — Candidate may be considered for a long position. Requires manual approval, signal creation, and decision creation. Does not create orders automatically.
HOLD / WATCH — Model does not recommend a new long entry at this time. Candidate is tracked but no signal is created.
REJECTED — Risk rules blocked the trade decision. Most common reason: MAX_POSITIONS_REACHED (portfolio is at the position limit). This is expected behavior, not a technical error.
SELL — Not generated in this candidate-entry workflow. A dedicated position-exit workflow based on open positions handles SELL decisions separately. Short selling is not implemented.
Decision = what the app recommends doing with a candidate | Rec = model recommendation | Conf = model confidence (0-1) | Return % = expected 5-day return | RS SPY = relative strength vs SPY over 20 days
